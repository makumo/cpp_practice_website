## 项目概述

这是一个用于练习C++语言的在线编程练习网站。网站提供编程题目，用户在指定区域编写C++程序，提交后系统会根据输入输出结果判断程序是否正确。

### 核心功能
- 显示C++编程练习题目
- 提供代码编辑器供用户编写C++程序
- 提交代码后自动编译和运行
- 根据预设的输入输出测试用例验证程序正确性
- 显示测试结果（通过/失败）

## iFlow 执行机制

**重要：以下机制必须严格遵守，不可遗忘**

### 1. 自动备份机制
- **触发时机**: 每次使用 `replace` 工具修改 .md 文件时
- **备份位置**: `docs/bak/` 目录
- **备份格式**: `原文件名.YYYYMMDD.NN.bak`
  - YYYYMMDD: 日期（如 20260308）
  - NN: 当日备份序号（01, 02, 03...）
- **执行方式**: 在修改文件前，先读取原文件内容，复制到备份目录
- **注意事项**: 备份文件会被 .gitignore 忽略，不提交到仓库

### 2. 命令超时机制
- **默认超时**: 5 分钟（300 秒）
- **超时处理**: 如果命令在超时时间内没有结果，取消执行并报告卡住的步骤
- **报告格式**: "命令执行超时，可能卡在: [步骤描述]"
- **例外情况**: 以下命令可以忽略超时规则
  - `npm install` - 安装依赖
  - `npm run dev` - 启动开发服务器
  - 大型构建操作
  - 已知可能长时间运行的任务

### 3. 敏感信息保护机制
- **敏感信息定义**:
  - API KEY
  - ACCESS TOKEN
  - 数据库连接字符串
  - 私钥
  - 证书文件
  - 密码
  - 密钥

- **显示脱敏**: 所有敏感信息在显示时按照"前4位+***+后4位"模式显示
  - 示例: `sk-1234567890abcdef` → `sk-1***cdef`
  - 示例: `mySecretPassword123` → `mySe***ssword`

- **日志脱敏**: 日志文件中不包含完整的敏感信息
- **命令历史脱敏**: 命令历史不记录敏感参数
  - 示例: `mysql -u root -p mypassword123` → `mysql -u root -p ****`
  - 示例: `curl -H "Authorization: Bearer token123"` → `curl -H "Authorization: Bearer ****"`

- **执行原则**:
  - 在显示任何可能包含敏感信息的内容前，先进行脱敏处理
  - 在记录命令历史时，自动脱敏敏感参数
  - 在日志输出时，确保不泄露敏感信息
